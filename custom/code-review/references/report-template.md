# Report Template

Use this structure for the branch review report.

# Key Findings

For each finding:

## [Severity] Title

File:
Problem:
Impact:
Suggested Fix:
Example: optional

# Summary

State branch intent, base and review refs, Jira/local context used, changed areas, risk level, and key concerns. Risk: Low, Medium, High, or Critical.

# Suggested PR Labels

List labels the eventual PR probably needs, plus any labels that may be questionable or context-dependent. Never imply labels were changed.

# Verification

List commands run and results. If verification was skipped or blocked, state why.

# Positive Notes

Include only substantive positives.

# Final Readiness Verdict

Choose one verdict from `references/verdict.md`, interpreting merge posture as pre-PR readiness.
